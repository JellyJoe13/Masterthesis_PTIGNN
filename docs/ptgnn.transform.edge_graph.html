ptgnn.transform.edge_graph.permutation_tree_selective.permutation_tree_transformation(data: Data, mol: Mol, k: int, tetrahedral_chiral: bool = True, chiral_center_selective: bool = False, chiral_center_select_potential: bool = True, remove_duplicate_edges: bool = False, cis_trans_edges: bool = False, cis_trans_edges_select_potential: bool = False, axial_chirality: bool = False, create_order_matrix: bool = True, multi_stereo_center_dia: bool = False, multi_stereo_center_dia_mode: int = 1, separate_tree: bool = False, add_cyclic_trees: bool = False, cyclic_tree_mode: str = 'complex', use_new_inv: bool = False) → Data[source]

Function with many bool control variables. Determines how the permutation tree is constructed and to which stereoisomer to be sensitive to.

Parameters:

  • data – vertex graph

  • mol – molecule which corresponds to vertex graph

  • k – number of neighbors to set into context. Not always necessary.

  • tetrahedral_chiral – Sensitive to tetrahedral chiral centers if enabled

  • chiral_center_selective – Sensitive to only marked tetrahedral chiral centers (and not by default to all nodes/atoms)

  • chiral_center_select_potential – Sensitive also to potential stereocenters (if selective is true)

  • remove_duplicate_edges – remove duplicate edges and reduce complexity - also often called reduction

  • cis_trans_edges – enable cis/trans trees (only selective)

  • cis_trans_edges_select_potential – enable cis/trans trees also for unmarked but potential cis/trans bonds

  • axial_chirality – Enable axial chirality - requires that rdkit has correct position values stored (is not true by default for version 2023.*.*)

  • create_order_matrix – whether or not to create the order matrix.

  • multi_stereo_center_dia – Whether or not do multiple stereo center enantiomer invariance. Not advisable as model can distinguish these stereoisomers (see master thesis) but difference is 10^(-4) which is too small.

  • multi_stereo_center_dia_mode – Mode that controls where to insert the MC trees. 0 creates new nodes that are not connected to remaining graph, 1 overwrites existing edges

  • separate_tree – Whether to separate tree - make simplest trees with only one internal node for each ptree for each node.

  • add_cyclic_trees – Add cyclic trees for graph - extra nodes, and more connections

  • cyclic_tree_mode (str) – Mode of the cylcic trees. Either complex, light or minimal - see master thesis

  • use_new_inv – Whether or not to separate C and Q types into P(Z,Z) & P(S,S) (if false) else P(Z2,Z2) & P(S2,S2)

Returns:

transformed graph

Return type:

torch_geometric.data.Data
